### 修复已延期/可能延期/正常筛选无效
- 根因：`ps`/`pe` 变量在 `if (searchFilter.date)` 块内声明，仅按状态筛选时 `pe` 为 `undefined`，`pe.month` 抛 TypeError
- 修复：将 `var ps`/`var pe` 提到 `forEach` 回调顶部，独立于日期筛选条件，始终计算

### 修复统计/筛选逻辑与 tooltip 不一致
- 根因：`updateStatsCards` 和 `getFilteredProjects` 使用静态 `delayDays`/`isDelayed` 字段判断「可能延期」/「正常」
- 而 `getProjectTooltip` 是动态算 `possibleDelay = actualStart日期 - projectStart日期`
- 导致「智能工厂全栈」(start 5/2, actualStart 5/3, delayDays:0) tooltip显示"可能延期1天"，但统计/筛选判为"正常"
- 修复：两处都改为动态计算 `overdueDays = max(0, ceil((today-endDate)/day))` 和 `possibleDelay = max(0, round((actualStart-startDate)/day) - overdueDays)`
- 结果：正常=2（智能客服、机器人视觉检测），可能延期=2（工业机器人装配线、智能工厂全栈）
