## SOP连接线修复（竖折连线间隔+贯穿线出头）

1. **单卡片节点连接线间隔消除**：`::after` 垂直线高度从 `1rem` 改为 `calc(2rem - 1px)`，延伸到 `::before` L折线 border-left 起始位置，消除1rem空白
2. **多卡片节点贯穿线不超出**：`.node-connector` 移除 `bottom: 0.5rem`，改为JS动态计算高度（`updateNodeConnectors()`），精确停在第二卡片L折线起点
3. **X坐标对齐修正**：`::before`/`::after` left 从 `calc(-0.75rem - 1px)` → `calc(-0.625rem - 1px)`，与圆圈中心X(1.875rem)精确对齐
4. **sibling卡片 `::after` 隐藏**：`display: none`，由贯穿线替代其垂直线功能

关键计算：圆圈中心X = 0.125rem + 0.75rem + 1rem = 1.875rem；卡片margin-left=2.5rem，所以伪元素left = -0.625rem

## SOP连接线二次修复（贯穿线Y坐标+JS计算修正）

1. **贯穿线起点修正**：`.node-connector` 的 `top` 从 `calc(0.5rem + 2rem)` 改为 `2rem`，因为 `0.5rem` 是 timeline 的 padding-top，而 connector 在 column 内（column 无 padding-top），不应包含 timeline 的 padding
2. **JS计算修正**：`updateNodeConnectors()` 中 `targetY` 去掉多余的 `cardPaddingTop`。`lastCard.offsetTop` 已经是卡片 padding edge 相对于 column 的位置（无 border 时），再加 `cardPaddingTop` 会导致贯穿线延伸过头 `0.75rem`
3. **有sibling时隐藏第一卡片 `::after`**：`.sop-column.has-sibling > .node-card:first-of-type::after { display: none }`，避免第一卡片自己的垂直线与贯穿线重叠冲突

修正后逻辑：
- 单卡片节点：`::after` 垂直线 + `::before` L折线（各自独立）
- 多卡片节点：一条 `.node-connector` 贯穿线替代所有垂直线段，两个卡片都只画 `::before` L折线
