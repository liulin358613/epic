# 2026-06-05 工作日志

## ai.html 侧边栏布局重构（文件夹 C）

### 修改内容
按照用户要求重构了 C/ai.html 的左侧侧边导航：

1. **三段式布局**：
   - `sidebar-fixed-top`：固定顶部区域，包含"我的秘书"和"项目经理"两个不可展开项
   - `sidebar-menus-area`：弹性中间区域，包含三个可展开菜单（我的人类伙伴、我的AI团队、组织栏）
   - `sidebar-login-bar`：固定底部个人信息栏，从绝对定位改为 flex 布局，确保永不移动/覆盖/折叠

2. **展开/收起核心行为**：
   - 三个可展开菜单展开时，下方菜单自动向下挤压
   - 当菜单被挤压到触底（个人信息栏），再展开其他菜单时自动收起所有已展开菜单，只保留当前最新点击的菜单
   - 展开区域内部变为独立可滚动区域（`.submenu-scroll-area`）

3. **滚动规则**：
   - 侧边栏整体不滚动（`overflow: hidden`）
   - 只有展开菜单的子菜单区域可滚动（`.submenu-scroll-area { overflow-y: auto }`）
   - 其他菜单保持不动

4. **CSS 变更**：
   - 移除 `sidebar-scroll-area` 样式（不再需要整体滚动区域）
   - `.sidebar-login-bar` 移除 `position: absolute`，改用 flex 布局
   - `.sidebar-menu` 新增 `expanded`/`collapsed` 类控制展开/收起状态
   - `.sidebar-menu.expanded` 默认 `flex-shrink: 0`（不收缩，按内容自动撑高）
   - `.sidebar-menu.expanded.overflow` 触底时 `flex: 0 1 auto`（允许收缩+滚动）
   - `.submenu-scroll-area` 默认不滚动（子级内容全部展示）；触底时才启用 `overflow-y: auto`
   - 新增 `updateOverflowState()` 函数，动态检测触底并切换 `.overflow` 类

5. **JS 重写（二次修复）**：
   - 用 Python 脚本精准替换整个 JS 段落（从 `// 团队成员折叠功能` 到 `// 项目列表点击事件处理` 之前）
   - 修复 Unicode 字符混用导致的 `sm`/`sm` 变量名错误
   - 删除不再使用的 `isOverflowing` 函数
   - `initMenuStates()`：初始化时三个菜单全部展开 → 不再收起触底菜单，而是加 `.overflow` 类让内容可滚动
   - `updateOverflowState()`：检测每个展开菜单**自身**的bottom是否超过loginBar的top，只有自身触底的菜单才加overflow类
   - 点击展开时：检测**当前菜单自身**是否触底（bottom > loginBar top），只有触底才收起其他菜单；未触底则不收起
   - 窗口 resize 时自动调用 `updateOverflowState()`

6. **触底判断修复（重要）**：
   - **问题**：点击展开组织栏时，即使组织栏并未触底，也会错误收起其他一级菜单
   - **根因**：展开当前菜单时先移除了所有其他菜单的 `.overflow` 类，导致这些菜单内容全部撑开，把组织栏推到超过个人信息栏位置，误判为触底
   - **修复**：展开当前菜单时**不移除其他菜单的 `.overflow` 类**，保持它们的收缩状态不变，这样测量当前菜单位置才是准确的；最后由 `updateOverflowState()` 统一重新计算
   - **二次修复**：将触底判断时机从"展开后"改为"展开前"——在菜单还是收起状态时检测其 header 的 bottom 是否已紧贴 loginBar 的 top，只有此时才算"触底"才收起其他菜单；展开后再测量会被内容撑高误导

7. **侧边栏逻辑完全重写（新规则）**：
   - **新规则**：展开菜单 → 下方菜单被挤压 → 触底的菜单固定展示+收起子菜单（不隐藏header） → 级联触底（组织栏触底个人信息栏，AI团队触底组织栏）
   - **点击触底菜单**：先收起所有已展开菜单 → 再展开当前菜单
   - 用 `at-bottom` CSS 类标记触底菜单，通过 `getBoundingClientRect()` 检测
   - 核心函数 `reconcileLayout(expandedMenu)`：从底部往上自动收起触底菜单（不收起当前展开菜单），标记 `at-bottom`，给超出边界的展开菜单添加 `overflow` 滚动
   - `recalcBottomState()`：收起菜单后重新评估触底状态
   - `initMenuStates()`：初始化全部展开后调用 `reconcileLayout`
   - resize 监听改为调用 `recalcBottomState`
